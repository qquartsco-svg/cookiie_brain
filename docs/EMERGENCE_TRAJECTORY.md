# 환경 → 자가순환 객체 창발 궤적

**목표**: 지구라는 공간에서, 중력·인력·척력이 아닌 **스스로 행동·움직임이 가능한 객체**가 나타나는 조건을 추적한다.

---

## 1. 궤적의 구조

```
[행성 상태공간] ──물리법칙 대입/연결──▶ [환경 설정] ──조건 충족──▶ [자가순환 객체 창발]
```

- **행성 상태공간**: 궤도 r(t), 스핀, 질량, 반지름, 대기 조성 …
- **환경 설정**: 물리 법칙이 하나씩 공간에 대입되고, 레이어끼리 연결됨
- **자가순환 객체**: 중력·인력·척력의 직접 영향이 아닌, **스스로 움직이는** 객체 (원시 대사, 세포, 뉴런 …)

---

## 2. 기어·부품 증가 원칙

지금부터 **기어·부품 수가 늘어난다**.

| 원칙 | 내용 |
|------|------|
| **순서** | 기어는 정의된 순서대로 넣는다. |
| **개념-폴더 정합** | 물리 개념과 코드 폴더·모듈이 1:1로 맞아야 한다. |
| **꼬이지 않기** | 새 기어는 기존 레이어 규칙을 깨지 않는다. (기어 분리, 관측자 모드) |
| **검증** | 각 기어 추가 후 최소 단위 테스트로 "맞음" 확인. |

---

## 3. 레이어·폴더 체계 (현재)

```
solar/
├── data/        ← 상태공간 데이터 (NASA 상수)
├── core/        ← 물리 코어 (중력, 궤도, 스핀, 세차, 해양)
├── em/          ← 전자기 환경 (광도, 자기장, 태양풍, 자기권)
├── atmosphere/  ← 대기 환경 (온실, 열적 관성, 수순환)
└── cognitive/   ← 인지 레이어 (관성 기억, Ring)
```

**규칙**:
- `core/`는 상위를 import하지 않음.
- `em/`, `atmosphere/`, `cognitive/`는 `core/`만 읽음. 궤도·물리 수정 없음.
- 새 레이어는 **위에 얹는** 형태. 기존 폴더 덮어쓰지 않음.

## 4. 단계별 기어 (순서)

### Phase 1–5: 외부 환경 ✅
- 중력, 궤도, 광도, 자기장, 태양풍, 자기권

### Phase 6a: 궁창 ✅
- 온실, 열적 관성, PT 경계, habitable 판정

### Phase 6b: 수순환 ✅
- 증발·응결·잠열, surface_heat_flux → SurfaceOcean

### Phase 7: 셋째날 — 땅과 바다 ✅
- SurfaceSchema: land_fraction, A_land, A_ocean
- effective_albedo → atmosphere 연동

### Phase 6c~: 환경 안정성·피드백 (예정)
- 대기 탈출, 수증기 피드백, ice-albedo, Outgassing, …

### Phase 7~: 물질 반응 동역학 (필수 경계)
- 현재: 환경 설정 완료. **수동 시스템** (ẋ = f(x) + F_external)
- 자가순환 객체는 **화학 반응 + 구배 + 확산 + 경계 형성** 없이는 나타나지 않음
- 최소 모델: Brusselator, Lotka-Volterra, BZ 반응 등
- 상세: [AUTONOMOUS_CYCLE_PHYSICS.md](AUTONOMOUS_CYCLE_PHYSICS.md)

## 5. 핵심 질문

> **공간의 환경 설정에서, 어떤 조건에서**  
> **자가순환하는 한 움직임을 가진 객체**  
> **(중력·인력·척력이 아닌, 스스로 행동·움직임이 가능한 객체)**  
> **가 나타나게 되는가?**

이 질문에 답하기 위해:
1. 물리 법칙을 **순서대로** 공간에 대입하고
2. 레이어를 **연결**하며
3. 상태공간 변화를 **관측**하고
4. **창발 조건**을 기록한다.
