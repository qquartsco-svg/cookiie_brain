# 에너지/엔트로피 관점 — 궁창·대홍수 시스템 파라미터 고정

**목적**: 종교 해석이 아니라 **대홍수(궁창 붕괴) 이벤트 전후** 시스템 파라미터가 어떻게 바뀌는지를 에너지/엔트로피 관점으로 고정해, 수명·항상성·경고·FSM 전이가 **동역학으로 자연스럽게** 따라오게 한다.  
**정의 → 수식(최소) → 이벤트 전/후 파라미터**로 딱 잡아두면 빌드업이 된다.

---

## 1. "에너지 총량" 정의 (개체 레벨)

에너지 총량을 **"평생 사용 가능한 유효 에너지/회복능력의 누적"**으로 정의:

- **E_cap**: 개체의 생애 에너지 용량(배터리/저장고)
- **P_in(t)**: 섭취/흡수로 들어오는 에너지(식량, 대사 입력)
- **C(t)**: 유지비(기본 생존), 스트레스 대응 비용, 수리 비용, 번식 비용 등 총 소모

**생존 조건**:

```
dE/dt = P_in(t) - C(t),   E(t) ≥ 0
```

**수명 T**: E가 0이 되는 시간.

---

## 2. 궁창 유무가 C(t)를 어떻게 바꾸는가

궁창은 "서사 장치"가 아니라 **환경 완충 장치(보호막)**.  
궁창 있을 때: 외부 손상/변동 작음 → 기본 유지비 C 낮고 안정.  
궁창 사라지면: 유지비 상승 + 변동성 증가 → 수명/항상성 다운그레이드.

```
C(t) = C_base + C_env(firmament) + C_repair(damage) + C_repro(fork)
```

궁창이 사라지는 순간 크게 바뀌는 것: **C_env**, **C_repair**.

---

## 3. 궁창 레이어를 "환경 파라미터"로 고정

궁창을 **보호 계수**로 모델링:

- **S(t) ∈ [0, 1]**: 보호막 강도(궁창 스케일). S=1 궁창 완전, S=0 붕괴(현재 지구에 가까움).
- **L_env(t)**: 외부 스트레스(자외선/방사선/기온변동/건조 등) 통합 환경 부하

```
L_env(t) = (1 - S(t)) · ( α_UV·UV(t) + α_ΔT·ΔT(t) + α_dry·D(t) + … )
```

유지비/수리비 전환:

```
C_env   = k_env · L_env(t)
C_repair = k_rep · max(0, L_env(t) - θ)
```

- 궁창 있음: S ≈ 1 ⇒ L_env ≈ 0 → 기본 유지비 낮음.
- 붕괴: S↓ ⇒ L_env↑ → 유지비/수리비 폭증 → 수명 하락.  
**"서사"가 아니라 동역학으로 전이.**

---

## 4. 빙하기/극지 얼음 — 같은 틀

궁창 붕괴 → 극지 얼음 → 빙하기 → 지금 다시 녹는 중(잔빙 = 남극·그린란드)을 시스템에 넣으려면, **얼음을 알베도 피드백 변수**로 둠.

- **I(t) ∈ [0, 1]**: 빙상(ice) 비율
- **알베도**: A(t) = A_0 + a_I·I(t)
- 에너지 균형(단순): dT/dt = β·((1−A(t))·Q⊙ − σ·T^4 − H(t))
- 빙상은 온도 임계로 생성/소멸: dI/dt = γ·( clip(T_freeze−T,0,∞) − η·clip(T−T_melt,0,∞) )

궁창 붕괴 = S(t) 하락 → T 변동/극지 냉각 커짐 → I(t) 형성 → I(t)가 다시 기후 변동성을 키워 L_env를 올리는 쪽으로 피드백 설계 가능.

---

## 5. 대홍수 이벤트 빌드업 — "3구간" 정의

스토리가 아니라 **시뮬레이터 상태 구간** 정의.

| 구간 | 이름 | S(t) | L_env / 비용 | 비고 |
|------|------|------|--------------|------|
| **0** | Pre-Flood (궁창 유지) | S = S_0 ≈ 1 | L_env 낮음, ΔT 작음 | 인간/에이전트는 **내부 요인**(번식/혼란/규칙위반) 때문에만 수명/상태가 갈림 → "120 vs 900" 구현 가능 |
| **1** | Transition (붕괴 진행) | S(t) 연속 하강 (몇 틱) | L_env 급증, 변동성 증가 | UnderWorld/Hades는 **경고 신호가 점점 강해지는** 구간 |
| **2** | Post-Flood (궁창 없음) | S ≈ 0 | I(t) 장기 형성 가능(빙하기/극지빙상) | 생명 유지비 구조적으로 높음 → 수명 하향 평준화. 이후 문명/기술 Δ_tech(t)로 L_env → L_env − Δ_tech 하면 "현대 수명 회복"도 같은 공식으로 설명 |

---

## 6. 수명 수렴(80~100) — 억지 없이 나오는 조건

"위에서 내려오고 아래에서 올라와서 수렴"이 시스템에서 자연스럽게 나오는 최소 조건:

- **아담 계열**: E_cap(t) 또는 회복 효율이 세대마다 감소(엔트로피/누적 손상).
- **일반 인류**: E_cap은 낮지만, 기술로 L_env를 줄여서 평균 수명 상승.
- **둘 다** 현대의 공통 외부 조건(S≈0, 기술 높음, 의료/위생) 아래에서 **비슷한 유지비 구조로 수렴**.

## 7. 다음 작업 (엔진 기준)

- **Layer 0 환경**: S(t), I(t) 같은 "궁창/빙상 상태변수"를 넣고, Homeostasis 비용식에서 L_env(t)로 유지비/수리비가 올라가게 연결.
- **UnderWorld (Hades)**: "S 하락/환경부하 증가"를 **감지해 경고만** 내보내게 유지.  
  → S(t), env_load를 **DeepSnapshot에 최소 침습으로 추가**.
- **IntegrityFSM**: 경고/손상/에너지 임계치가 넘을 때만 상태 전이(다운그레이드)가 일어나게 유지.

이렇게 하면 "대홍수 이벤트"가 **스토리 트리거가 아니라 환경 파라미터의 급변**으로 들어가서, 그 다음 레이어가 다 동역학으로 따라온다.
