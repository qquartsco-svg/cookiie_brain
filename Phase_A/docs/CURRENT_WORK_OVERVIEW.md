**당신이 알아야 할 것만 적었습니다.**

## 1. 우리가 하려는 일 (목표)

- **상태 공간**에 **에너지 우물** 하나가 있음 (최소 에너지 포텐셜 덩어리).
- 지금은 그냥 **우물 바닥으로 떨어지기만** 함 (curl=0, 돌지 않음).
- **하고 싶은 것**: 그 우물 안/주변에서 **자전**처럼 **돌게** 만드는 것.

**한 줄**: 우물에 고인 걸 **자전** 시키기.

---

## 2. 그걸 위해 "무슨 작업"을 했는지

### (1) 수식 정리 (이미 되어 있던 것)

- 합성 필드: **g(x) = -∇E(x) + R(x)**  
  - 앞쪽: 우물로 끌어당기는 힘  
  - 뒤쪽: **회전 성분** (R은 반대칭 행렬로, r·R=0 만족 → 에너지 주입/빼기 없음)
- **R(x) = ω J (x - x_pole)** 형태로 두고, **단순 자전**부터 하기로 함.

### (2) 코드에 넣은 것 (내가 한 작업)

- **PotentialFieldEngine**  
  - "필드" 계산할 때 **회전 항 R(x)** 를 더할 수 있게 옵션 추가.  
  - 옵션 안 넣으면 **예전과 똑같이** 동작 (호환 유지).
- **CookiieBrain**  
  - 설정으로 **"자전 켜기"** 플래그 추가 (`enable_phase_a`).  
  - 켜면: 회전 중심(pole), 세기(strength), 방향 넣어서 **R(x)** 만들어서 엔진에 넘김.
- **테스트/검증용 스크립트**  
  - **우물 + 자전** 한 번 돌려보는 스크립트  
  - **부호 맞는지**, **r·R=0 맞는지** 수치로 확인하는 스크립트  

→ **정리**: "우물만" → "우물 + 자전"으로 **바꿀 수 있게** 만든 것.  
당신이 **설정만 바꿔서** 자전 모드로 켜거나 끌 수 있음.

## 3. "지금 무슨 작업"이라고 부르는 것 = 위 (2) 전부

- **작업 1**: PotentialFieldEngine에 회전 항 넣을 수 있게 확장  
- **작업 2**: CookiieBrain에서 그걸 켜는 설정 추가  
- **작업 3**:  
  - "우물 + 자전" 한 번 돌려보는 테스트  
  - "부호 / r·R=0" 수치 검증 스크립트  
- **작업 4**:  
  - 문서/체크리스트에 **지금 상태**(코드 통합 완료, 검증 스크립트 있음) 맞춤  
  - **피드백**(감쇠 없음, 2D 전용 등) 반영해서 "뭘 해야 하는지" 정리  

**당신이 할 일은 아직 없음.**  
지금 단계는 "시스템이 자전 모드로 동작할 수 있게 만들고, 검증 도구까지 넣어 둔 상태"입니다.
