# JOE 역방향 모드: 현재 지구 → 초기 상태 추적

조(JOE) 엔진은 두 방향으로 쓰인다.

1. **정방향 (탐사)**: 우주·행성 물리 입력 → Feature Layer → 스냅샷 → Aggregator → planet_stress, instability, habitability.  
   “이 행성이 테라포밍 가능한가?”

2. **역방향 (추적)**: **현재 지구의 관측값**을 입력하면, 동역학·물리 법칙에 따라 **초기 상태(태초에 가까운 조건)**를 추정할 수 있어야 한다.

---

## 설계 원칙 (필수)

- **역추적 = “역적분”이 아님.**  
  시간을 거꾸로 적분해 “유일한 초기 상태”를 복원하는 것은 일반적으로 **ill-posed**(해가 여러 개이거나 불안정)이다.

- **역추적 = “추정/동화(assimilation)”로 설계**
  - 목표: “초기상태 하나”가 아니라  
    **초기 후보 분포/범위(가능한 초기 조건들의 집합)** + **MAP 추정치(최적 후보)**.
  - 방법:  
    1. **정방향(Forward) 거시 동역학 모델**을 정의하고,  
    2. **관측(현재 지구 데이터)**에 맞게 초기 파라미터를 **최적화/필터링/샘플링**한다.

- JOE는 “물리 시뮬레이터”가 아니라 **평가기(Observer)**이므로, 역방향도 “전체 역적분”이 아니라 **거시 스냅샷 수준의 역추정**으로 한정한다.

- **정보 추적 엔진**(Hippo/메모리/lineage)과 결합 시:  
  “t=0(초기) ~ t=T(현재)” 상태 시퀀스를 주면 JOE가 시점별 스냅샷을 평가하고, t=0을 “초기 상태”로 정의·연결할 수 있다.  
  **역사 추적 엔진(HistoricalDataReconstructor)** 연동 가능성은 [JOE_HISTORICAL_RECONSTRUCTOR_INTEGRATION.md](JOE_HISTORICAL_RECONSTRUCTOR_INTEGRATION.md) 참고.

## 두 모드 정의

### Forward 모드 (정방향)

- **입력**: 초기 거시 상태 x0 (또는 그에 대응하는 스냅샷).  
  예: 조 레벨 상태벡터 [M, R, ω, ε, e, W_total, W_surface, sigma_plate, P_w, …]
- **출력**: 시간에 따른 macro snapshot `snap(t)` 또는 `x(t)`.
- **역할**: 역추적을 가능하게 하는 “정방향 거시 동역학”이 있어야 Inference가 성립한다.

### Inference 모드 (역추적)

- **입력**: 관측 `y_now` (현재 지구 데이터, 또는 엔진이 만든 스냅샷).
- **출력**:
  - `x0_hat`: 최적 초기 후보 (MAP 추정),
  - `confidence` / `range`: 가능 범위,
  - 또는 **초기 후보 집합** `list[snapshot]`.
- **구현 방법(거시 모델에 현실적으로 맞는 것)**:
  - (단순) 파라미터 스윕 + 목적함수 최소화
  - (중간) 진화전략 / 베이지안 최적화
  - (정석) EKF / UKF / Particle Filter 등 동화(assimilation)

**핵심**: 역추적은 “정답 하나”가 아니라 **조건을 만족하는 초기 후보들의 집합을 좁혀가는 엔진**으로 설계한다.

---

## 역방향 연산 형태 (개념)

- **Feature Layer 역산**:  
  현재 스냅샷의 `g_surface_norm`, `v_escape_norm`, `S_rot` 등에서 **가능한 (M, R, ω)** 조합을 유도 → 일대다이므로 **후보 집합**.
- **Aggregator는 그대로**:  
  역으로 얻은 “초기 스냅샷 후보”를 정방향으로 다시 넣어 stress/instability가 일관한지 검증.
- **시간 축과 결합**:  
  정보 추적 엔진이 시점별 스냅샷을 주면, JOE는 각 시점을 평가하고 t=0을 “초기 상태”로 사용.  
  현재 지구만 주어지면: “현재 스냅샷 → 역추정 후보 → 그중 stress가 낮은 것 = 초기로 가정 가능한 조건”.

## 구현 위치 및 계약

- **인터페이스**: `joe/forward_inference.py`  
  - `forward_dynamics(x0, t_span, ...)` → `list[snapshot]` 또는 시계열  
  - `infer_initial_candidates(y_now, constraints=None, ...)` → `x0_hat`, `candidates`, `confidence/range`
- **정보 추적 엔진**:  
  기존 `analysis/`, `hippo/`, `solar/.../lineage` 등과의 연동은 “시점별 스냅샷”을 JOE에 넘기는 어댑터로 정리.

---

## 요약

- **정방향**: 물리 → 스냅샷 → 평가 (현재 구현).
- **역방향**: 현재 지구 스냅샷 → **(추정/동화)** → 초기 후보 분포/범위 + MAP; 필요 시 정보 추적 엔진의 시간 축과 결합.
- JOE는 계속 **거시 스냅샷 입출력**만 담당하고, 역방향도 **스냅샷 수준의 역추정**으로 한정해 재현 가능한 물리 리포트로 유지한다.
